# Changelog

## v1.2.0 (2026-06-18)

### Changed

- **BR 1本の粒度・長さに歯止めを追加** — 「1ルール＝1判断、原則1〜2文。3文以上またはおおむね150字を超えたら黄信号」という基準を `business_rules_guide.md`・`business_rules_driven_development.md` に明文化。長い BR は複数判断の同居・経緯の漏れ・仕様の混入を疑い、分解 / 経緯を proposal へ移動 / 仕様を機能仕様・HLD へ除外 のいずれかで対処する。150字はハード上限ではなく再点検の合図
- **禊（misogi）の事前審査に「粒度・長さ検査」を追加** — サブエージェントが BR 候補の長さ過大を「粒度過大」として指摘するようにした

## v1.1.1 (2026-06-14)

### Changed

- **proposal 受け付けスキルのケイパビリティ名指定を廃止** — `miko.quick_impl` / `miko.split_proposal` / `miko.speckit.specify` でケイパビリティ名だけを渡したときに最新プロポーザルを自動選択する挙動を削除。プロポーザルファイルのパス直接指定のみに統一した

## v1.1.0 (2026-06-10)

### New

- **出力言語の選択に対応（日本語 / English）** — インストール時に出力言語を選択できるようにした。選択は `.miko/config` に保存され、対話と生成ドキュメントの言語が統一される。`MIKO_LANG=en` 環境変数でプロンプトを省略可能
- **英語版 tone_guide（`tone_guide.en.md`）を追加** — 英語の対話トーンに加え、生成ドキュメントの正準見出し対訳表を含む。スキル・ガイド・examples は日本語のまま、出力のみ英語化する方式
- **英語版 README（`README.en.md`）を追加** — 両 README の冒頭に言語リンクを設置

### Changed

- tone_guide に「出力言語」セクションを追加し、適用範囲を対話のみから全出力（生成ドキュメント含む）に拡大
- speckit 系 7 スキルに tone_guide への参照を追加（従来は未参照だった）
- upgrade.sh が `.miko/config` の言語設定を読んで tone_guide を解決するようにした（config がない既存インストールは日本語扱い）
- install.sh / upgrade.sh 自体のメッセージも選択言語で表示するようにした
