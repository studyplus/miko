# Changelog

## v1.3.1 (2026-06-23)

### Changed

- **既存 BR の構造を保持する方針を明文化** — v1.3.0 で導入したビジネスポリシー層・2.1/2.2 分割は新規ケイパビリティ向けであり、既存 `business_rules.md` には遡及適用しない旨を `business_rules_guide.md` と `skills/miko.propose/SKILL.md` に追記。既存 BR の構造もルール本文も、ユーザーから明示的な移行依頼がない限り変更しない（文字数や語彙規律など現在基準への合わせ込みも含めて）

## v1.3.0 (2026-06-23)

### New

- **ビジネスポリシー（ビジネス方針）層を導入** — ケイパビリティに通底する価値判断・指針を `## ビジネスポリシー（ビジネス方針）` セクション（`POL-{連番}`）として business_rules.md に新設。判定は「客観性」テスト（任意のケースで客観的に判定できるか）— Yes ならルール、No ならポリシー。コードから抽出する際にルールに紛れ込みがちな価値判断を意識的に分離する層。ルールとポリシーは整合関係であり、機械的な導出関係ではない（ルール → ポリシーのバックリンクは持たない）
- **ビジネスルールをドメイン / システムに分割** — ビジネスルール・カタログを `### 2.1 ドメインビジネスルール`（紙の運用でも成り立つもの）と `### 2.2 システムビジネスルール`（非同期・冪等性・整合性回復など、システム前提のもの）の 2 セクションに分けて配置。判定は「紙運用」テスト。1 ルール内にドメイン判断とシステム判断が混在することを避ける

### Changed

- `business_rules_guide.md`・`business_rules_driven_development.md`・`examples/business_rules.md`・`tone_guide.en.md` を上記 2 点に追従して更新

## v1.2.0 (2026-06-18)

### Changed

- **BR 1本の粒度・長さに歯止めを追加** — 「本文がおおむね150字超」または「独立した判断が3つ以上」を詰め込みすぎのサイン（黄信号）とする基準を `business_rules_guide.md`・`business_rules_driven_development.md` に明文化。文の数に固定の上限は設けない。黄信号が出たら 分解 / 経緯を本文から落とす / 仕様を機能仕様・HLD へ除外 のいずれかで対処する。150字はハード上限ではなく再点検の合図
- **禊（misogi）の事前審査に「粒度・長さ検査」を追加** — サブエージェントが BR 候補の長さ過大・判断の同居を「粒度過大」として指摘するようにした

## v1.1.1 (2026-06-14)

### Changed

- **proposal 受け付けスキルのケイパビリティ名指定を廃止** — `miko.quick_impl` / `miko.split_proposal` / `miko.speckit.specify` でケイパビリティ名だけを渡したときに最新プロポーザルを自動選択する挙動を削除。プロポーザルファイルのパス直接指定のみに統一した

## v1.1.0 (2026-06-10)

### New

- **出力言語の選択に対応（日本語 / English）** — インストール時に出力言語を選択できるようにした。選択は `.miko/config` に保存され、対話と生成ドキュメントの言語が統一される。`MIKO_LANG=en` 環境変数でプロンプトを省略可能
- **英語版 tone_guide（`tone_guide.en.md`）を追加** — 英語の対話トーンに加え、生成ドキュメントの正準見出し対訳表を含む。スキル・ガイド・examples は日本語のまま、出力のみ英語化する方式
- **英語版 README（`README.en.md`）を追加** — 両 README の冒頭に言語リンクを設置

### Changed

- tone_guide に「出力言語」セクションを追加し、適用範囲を対話のみから全出力（生成ドキュメント含む）に拡大
- speckit 系 7 スキルに tone_guide への参照を追加（従来は未参照だった）
- upgrade.sh が `.miko/config` の言語設定を読んで tone_guide を解決するようにした（config がない既存インストールは日本語扱い）
- install.sh / upgrade.sh 自体のメッセージも選択言語で表示するようにした
