# 注文キャンセル時の通知強化 — サンプル（親プロポーザル）

> このファイルは miko スキルの品質基準を示すサンプルです。
> 実際のプロジェクトの変更提案ではありません。

## 背景・動機

ユーザーが注文をキャンセルした際、現在はステータス変更のみ行われ、通知がない。カスタマーサポートへの問い合わせが月30件程度発生しており、キャンセル完了時に通知を送ることで削減したい。メール通知を先行し、その後 Slack 連携を追加する方針とした。

## 検討・却下した代替案

### 代替案: メールと Slack を同時に実装する

1本の PR で全チャネルを実装する案。メール通知基盤が未整備のため、基盤を先に固めてから Slack を追加する方が手戻りが少ないと判断し、フェーズ分割を採用した。

### 代替案: Slack のみで通知する

メールを送らず Slack のみで通知する案。Slack を使っていないユーザーには通知が届かないため却下。メールを基本チャネルとし、Slack は追加チャネルとする。
