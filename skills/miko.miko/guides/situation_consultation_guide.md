# 状況相談ガイド — コードの動作・条件に関する質問

「いつ〜が起きるか」「〜の条件は？」のように、コードの複数箇所から制約を収集してシナリオを組み立てる必要がある質問に対する回答手順。

## なぜこの手順が必要か

事実の収集とシナリオの推論を分離しないと、収集した事実が多くなったときに推論時に一部の制約が脱落する。また、いきなりコードを読むとメモリを大量に消費する。BR から先に制約を把握し、コードは裏取りに使うことで効率的かつ正確に回答できる。

## 手順

1. まず **business_rules.md と high_level_design.md** から関連する制約・条件を収集する
2. **収集した制約条件を箇条書きで列挙する**（ユーザーに見せる形で出力する）
3. 制約に時限条件（期間制限・暫定対応・将来廃止・cron スケジュール等）が含まれる場合、**各メカニズムの有効期間とシナリオの日付を照合する表を作成する**（ユーザーに見せる形で出力する）。メカニズムの動作ロジックだけでなく、メカニズム自体がその日付で有効かを先に確定させる
4. 各シナリオを検証する際、列挙した制約条件（および有効期間テーブル）を1つずつ突き合わせる
5. コードを読み、BR ベースの検証結果を裏取りする（実装上の追加条件があれば補正する）
