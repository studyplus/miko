## このスキルの役割

`miko/system_high_level_design.md` がコードベースの現状と乖離していないかチェックし、必要に応じて更新する。

`miko.setup` が初回生成を担うのに対し、このスキルはその後のメンテナンスを担う。ディレクトリ構成の変化、新しいレイヤーやAPI構造の追加、フレームワークのアップグレード等を検出して反映する。

**出力:** `miko/system_high_level_design.md`（更新がある場合のみ）

- `miko/system_high_level_design.md` が存在するか確認する
- **存在しない場合はエラーとして中止する:**
  > ⛩️  `miko/system_high_level_design.md` が見つかりません。
  > 初回セットアップは `/miko.setup` をお使いください。

### 2. ガイドと参考資料の読み込み

以下のファイルを読み込む。

**指示書（必須）:**
- `.miko/guides/tone_guide.md` — ユーザーとの対話スタイル。**このファイルの口調・絵文字・出力言語ルールに従うこと**

**実例（品質の基準）:**

**現行ファイル:**

### 6. 【確認】更新内容の確認

検出した乖離と更新案を、セクション別（ディレクトリ構成/レイヤー構成/API構造/着目点/ケイパビリティ一覧）にユーザーに提示し、承認を取る。変更がないセクションは省略する。

### 7. system_high_level_design.md 更新

確認を踏まえ、既存ファイルをベースに差分を反映する。

**保持するもの:**
- 既存の記述は消さない（変更が必要な場合は改訂）
- システム概要・テナント構造は変更しない（変更が必要な場合はユーザーに確認済みの内容のみ反映）

**更新するもの:**
- ディレクトリ構成
- レイヤー構成と責務
- API 構造（変更がある場合）
- ファイル命名規約（変更がある場合）
- ビジネスルール抽出の着目点（変更がある場合）
- ケイパビリティ一覧（miko/ 配下のディレクトリとの整合）

### 8. 完了報告

更新した項目（ディレクトリ構成・レイヤー構成・API 構造・着目点・ケイパビリティ一覧）の変更件数をサマリーテーブルで提示する。次のアクションとして `/miko.catchup` をご案内する。
