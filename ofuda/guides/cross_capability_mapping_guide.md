# 他ケイパビリティ実装マッピング更新ガイド

このファイルは、`business_rules.md` の実装マッピングを最新の実装に合わせて更新する手順書である。この内容に正確に従うこと。

## 前提

呼び出し元から以下が渡される:
- ベースブランチ名（`git diff` の比較基準）

## 表記規約

実装マッピングは `.miko/guides/business_rules_guide.md` の「表記規約」セクションに従った表記で記録されている。検索クエリもこの表記に合わせて組み立てる必要がある。

## 手順

### 1. diff を分析して検索対象を特定する

`git diff <base-branch>...HEAD` を実行し、削除側の diff（`-` で始まる行）を分析する。テストファイル・ドキュメントファイルの変更は無視する。

以下を収集する:
- 削除されたスコープ（クラス、モジュール等）とそのメンバ（メソッド・定数）
- リネームされたスコープやメンバ（旧名側）

該当するものがなければ、ここで終了する。

### 2. 表記規約に従った検索クエリを組み立てる

ステップ 1 の結果を、`.miko/guides/business_rules_guide.md` の表記規約に従った形式に変換する。

### 3. 実装マッピングを検索

ステップ 2 で組み立てた各検索クエリについて、`miko/*/business_rules.md` を Grep で検索する。

ヒットしなければ、ここで終了する。

### 4. 実装マッピングの更新

ヒットした箇所について、実態に合わせてマッピングを更新する。

**更新パターン:**
- **リネーム**: 旧名 → 新名に置換する
- **削除**: マッピングから除去する。代替の実装先がわかる場合は新しいマッピングに置き換える

**注意:**
- ルール本文（`<details>` の外）は変更しない。実装マッピングのみを更新する
- 更新の確信が持てない場合は、該当箇所に `<!-- TODO: 要確認 -->` コメントを付与する

## 検出限界

このガイドの手順で検出できるのは、スコープやメンバの**削除・リネーム**のみである。以下のような変更はこの手順では検出できない:

- メソッド名は変わらないが、主要ロジックが別メソッドに移動した場合（メソッド分割等）
- 既存メソッドの責務が大きく変わった場合

これらは実装者が変更時に気づいて対応する必要がある。

## 完了後の報告

更新した内容を以下の形式で報告する:

```
### 実装マッピング更新

| ケイパビリティ | ルール ID | 変更内容 |
|---|---|---|
| {capability} | {RULE-ID} | {旧} → {新} |

更新なしの場合: 「他ケイパビリティの実装マッピングに影響はありませんでした」
```
